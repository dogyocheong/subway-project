---
name: Indoor nav JSON path resolution
description: How to correctly resolve the dynamic indoor-nav-data.json path from compiled server code.
---

The API server is compiled by esbuild from src/ to dist/. When the compiled server runs, `import.meta.dirname` resolves to `dist/`, not `src/routes/`.

**Rule:** For any JSON file that is written at runtime (not statically imported), use `process.cwd()` instead of `import.meta.dirname`:
```ts
const p = path.join(process.cwd(), "src/routes/indoor-nav-data.json");
```
The server starts with CWD = `artifacts/api-server/`, so this correctly reaches the live file.

**Why:** Static JSON imports (toilet-data.json etc.) are bundled by esbuild into dist/index.mjs and work fine. But indoor-nav-data.json is generated by a background analysis script and updated continuously — it cannot be bundled at build time.

**How to apply:** Any route that reads a file written by an external process should use `process.cwd()` relative paths rather than `import.meta.dirname`.
